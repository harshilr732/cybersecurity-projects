# Practical 10: HTTP Traffic Analysis (Cleartext Communication)

## Aim
To analyze HTTP traffic and understand its insecurity.

## Procedure
1. Start capture  
2. Visit:http://testphp.vulnweb.com
3. Perform login  
4. Apply filter:http

## Observations
- HTTP GET and POST visible  
- Credentials visible in plaintext  
- Headers readable  

## Analysis
- No encryption  
- Data transmitted in cleartext  

## Security Perspective
- Vulnerable to:
  - Credential theft  
  - Session hijacking  
  - MITM attacks  
## Conclusion
HTTP is insecure; sensitive data is exposed and should be replaced with HTTPS.
