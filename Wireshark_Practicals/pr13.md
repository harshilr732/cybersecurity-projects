# Practical 13: Detection of Port Scanning (TCP Behavior)

## Aim
To detect reconnaissance using TCP patterns.

## Procedure
Apply filter:

tcp.flags.syn == 1 && tcp.flags.ack == 0


## Observations
- Multiple SYN packets  
- Incomplete handshakes  
- No data transfer  

## Analysis
Normal:
- SYN → SYN-ACK → ACK  

Scanning:
- SYN without completion  
- Multiple ports targeted  

## Security Perspective
Indicators:
- High SYN count  
- Multiple destination ports  

## Conclusion
Incomplete TCP connections indicate scanning activity.
