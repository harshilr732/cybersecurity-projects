# Practical 14: Brute Force Pattern Detection

## Aim
To identify brute force login attempts using traffic patterns.

## Observations
- Repeated login requests  
- Same endpoint accessed  
- Rapid request rate  

## Analysis
Normal:
- Few attempts, time gaps  

Brute force:
- High frequency  
- Repetition  
- No successful session  

## Security Perspective
Indicators:
- Multiple attempts from same IP  
- Continuous failures  

## Conclusion
Brute force is detected via patterns, not single packets.
