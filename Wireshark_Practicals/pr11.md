# Practical 11: HTTPS / TLS Traffic Analysis

## Aim
To analyze HTTPS traffic and understand what is visible versus encrypted.

## Procedure
1. Start capture  
2. Visit:https://example.com
3. Apply filter: tls

## Observations
- TLS handshake visible:
  - Client Hello  
  - Server Hello  
- Certificate and cipher suite visible  
- Application data appears encrypted  

## Analysis
- Handshake is visible (metadata)  
- Payload is encrypted  

## Security Perspective
Visible:
- IP, port (443), TLS version, SNI  

Not visible:
- Credentials, page content  

## Conclusion
HTTPS hides sensitive data but still exposes metadata useful for analysis.
