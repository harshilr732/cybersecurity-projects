# Practical 12: DNS Traffic Analysis and Abuse Detection

## Aim
To understand DNS behavior and identify misuse patterns.

## Procedure
1. Start capture  
2. Visit multiple websites  
3. Apply filter: dns


## Observations
- Multiple DNS queries per site  
- Query–response pattern observed  

## Analysis
- DNS resolves domain → IP  
- Uses UDP, no handshake  

## Security Perspective
Suspicious patterns:
- Random domains  
- High query volume  
- NXDOMAIN spikes  

### Possible abuses:
- DNS tunneling  
- C2 communication  

## Conclusion
DNS patterns reveal behavior even without payload visibility.
